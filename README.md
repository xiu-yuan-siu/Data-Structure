## 🎯 项目介绍

本项目是基于 **严蔚敏《数据结构（第三版）》** 的完整教材级代码实现，覆盖全书 **12 章** 全部核心内容，第一轮实现以 **C 语言为主** ，第二轮复习时(如果有空)会以 **C++ 为主** 使用 STL 重写，本意打算以 **C 语言** 基于原书严格实现**书中伪代码** ，使用 **C++ 特性(模板、类)** 方便对比其优势(`new`分配内存失败自动保持错误等)，以及熟悉C或C++的人都有看得懂的代码(刚开始我只学了C++，后面就跟着书学了 `malloc`, `free`, `struct`, `union`)，但是后面发现我学的太慢了，故而只实现了**前2、3章**，现在发现C/C++殊途同归，懂一个的肯定看得懂另一个，所以如果有空(保研成功!)就用 STL 实现。

### 为什么要做这个项目？

大多数数据结构学习者面临的困境是：

- 📕 **教材代码太抽象**——伪代码 + 片段，无法直接运行
- 💻 **网络代码对原书学习者不友好**——并未完全复现原书伪代码，对比看割裂感强
- 🧪 **缺少中文注释**——不理解每一步"为什么这样写"
- ⚡ **缺少运行结果**——无法评估自己写的代码是否正确
- 📝 **408 考点不突出**——不知道哪些是考试重点

这个项目彻底解决了以上问题：**每一行代码都来自教材思想，每一个函数都有中文注释，每一个边界条件都有专门讨论。**

## ✨ 项目特点

### 📝 1.极其详细的中文注释

例如：
```c
// 下面两步不能交换位置
s->next = p->next;
p->next = s;
```

每一处"先做什么、再做什么、为什么不能反过来"都有注释说明。

### 📖 2.严格基于严书教材源代码

严格遵循严蔚敏教材的 **ADT 定义** 和 **算法描述**，使用 `Status` 返回值、`ElemType` 泛型、`typedef` 重定义等经典教材写法，可对比原书伪代码，**本代码绝对还原(有的需要用循环实现赋值等小改变也注明了)且可运行**。

### 🧪 3.测试覆盖(前七章)

**前七章**均有测试代码接口(感谢Qwen-3.6-plus，Gemini-3.1，kimi-2.6)，我自己测试**均可运行**，读者可以自己修改。

### 🚫 3.不依赖 STL

> 所有数据结构**从零手写**——顺序表用 `malloc` 动态数组，链表用手动 `malloc/free` 管理结点，栈和队列全部自主实现。

这意味着你能看到**最底层的内存操作**，而不是被 STL 封装掩盖。

### 🧠 5.强调算法思想与边界条件

每个算法都包含：

- ✅ **核心思想**（为什么这样设计）
- ✅ **边界条件分析**（空表、满栈、越界）
- ✅ **408 考点标注**（高频易错点）
- ✅ **性能分析**（时间复杂度 / 空间复杂度）
- ✅ **多种实现对比**（C 版 vs C++ 版，递归 vs 非递归）

### 最容易错的点

> 💡 **指针问题：** C 语言中链表操作的二级指针（`LinkList *L`）是初学者的第一大障碍  
> 💡 **边界条件：** 循环队列判满（牺牲一个空间 vs 增设标志位）是高频易错点  
> 💡 **KMP next 数组：** `next[1]=0` 的初始化、`j=next[j]` 的回溯逻辑是最大难点  
> 💡 **线索二叉树：** `LTag`/`RTag` 标志位区分"指针"与"线索"是核心

### 第2章：线性表

**核心内容：** 线性表的两种存储结构 —— 顺序表 vs 链表。

#### 顺序表

📌 **408 考点：**
- 动态扩容策略：`LIST_INIT_SIZE` + `LISTINCREMENT` 增量
- 插入/删除操作的平均移动次数 = n/2（前半部分）或 (n-1)/2（后半部分）
- **随机存取**特性：O(1) 访问任意位置

#### 单链表

📌 **巧妙实现：**
- **头结点**：带头结点的链表统一了空表与非空表的操作，无需在插入/删除时特殊处理头指针
- **头结点 data 域存长度**：`L->data` 存储链表长度，`ListEmpty` 可直接通过 `L->data==0` 判断
- **二级指针**：`InitList(LinkList *L)` 中需要修改头指针的值，所以传指针的指针
- **插入两步不能交换**：`s->next = p->next; p->next = s;` —— 必须先连后断

```c
// 下面两步不能交换位置
s->next = p->next;   // 先让新结点指向后继
p->next = s;          // 再让前驱指向新结点
// 交换则丢失 p->next 的原有信息！
```

#### 循环链表

📌 **巧妙实现：**
- 尾指针 (`rear`) 而非头指针：从尾指针可以 O(1) 访问头结点和尾结点
- **循环条件不同**：`while (p != head)` 而非 `while (p != NULL)`

#### 双向链表

📌 **巧妙实现：**
- `prior` 和 `next` 双链，插入/删除需要同时修改两个方向的指针
- 双向循环链表：头结点的 `prior` 指向尾结点，尾结点的 `next` 指向头结点

#### 多项式的表示与相加

📌 **教学价值：**
- 链表在实际问题中的应用典范
- 按指数降序排列 + 合并同类项 = 经典算法设计思想

### 第3章：栈与队列

**核心思想：** 栈是"后进先出（LIFO）"，队列是"先进先出（FIFO）"。

#### 顺序栈

📌 **巧妙实现：**
- 动态扩容：`realloc` 在容量不足时自动扩展（`STACKINCREMENT`）
- `base` 指向栈底，`top` 指向栈顶元素的下一个位置
- `StackEmpty`：`base == top` 为空

#### 链栈

📌 **教学价值：**
- 相比顺序栈无需预估容量，更适合理解动态内存结构
- 链栈的入栈 = 头插法，出栈 = 删除首元结点

#### 栈的应用 —— 经典 4 例

| 应用 | 核心思想 | 408 频率 |
|------|----------|----------|
| **数制转换** | 除基取余法，先求的余数后输出（LIFO） | ⭐⭐⭐ |
| **括号匹配** | 左括号入栈，右括号匹配则出栈，最后栈空则合法 | ⭐⭐⭐⭐ |
| **迷宫求解** | 回溯法：试探→入栈→成功/失败→回溯出栈 | ⭐⭐⭐ |
| **表达式求值** | 算符优先法：操作数栈 + 运算符栈 + 优先级比较 | ⭐⭐⭐⭐ |

**📌 表达式求值的巧妙之处：**
- 使用**两个栈**（OPTR 运算符栈 + OPND 操作数栈）
- 算符优先级表是核心数据结构
- `#` 作为表达式结束标志，与栈底的 `#` 相遇表示计算完成

#### 递归 —— 阶乘 & 汉诺塔

📌 **教学价值：**
- 递归的本质：**函数调用栈**
- 汉诺塔的 `Hanoi(n, A, B, C)`：将 n 个盘子从 A 经 B 移到 C
  - 先把 n-1 个从 A 经 C 移到 B
  - 再把第 n 个从 A 移到 C
  - 最后把 n-1 个从 B 经 A 移到 C
- 理解递归 = 理解"规模缩小 + 递归出口"

#### 循环队列

📌 **巧妙实现：**
- **牺牲一个空间**判满：`(rear+1) % MAXSIZE == front`
- 模运算实现循环：`rear = (rear+1) % MAXSIZE`
- 队列长度公式：`(rear - front + MAXSIZE) % MAXSIZE`

📌 **408 易错点：**
- 队空条件：`front == rear`
- 队满条件：`(rear+1) % MAXSIZE == front`
- 入队：`rear = (rear+1) % MAXSIZE`
- 出队：`front = (front+1) % MAXSIZE`

#### 核心：KMP 算法 ★★★

这是 408 数据结构中**最难以理解**的算法之一，选择题几乎每年 1 道。

**📌 核心思想：**
- 暴力匹配（BF）在失配时主串 i 回溯，模式串 j 重置为 1，时间复杂度 O(m×n)
- KMP 的核心：**主串 i 不回溯**，只移动模式串 j
- 关键在于理解 `next[j]` = **当第 j 位失配时，j 应该跳转到哪里**

**📌 next 数组的本质：**
- `next[j]` = 模式串中前 j-1 个字符的**最长相等前后缀长度 + 1**
- `next[1] = 0`（特殊标记，表示 i 进位、j 复位）

```c
void get_next(SString T, int next[]) {
    int i = 1;
    next[1] = 0;
    int j = 0;
    while (i < T[0]) {
        if (j == 0 || T[i] == T[j]) {
            ++i; ++j;
            next[i] = j;  // 巧妙：最长相等前后缀 + 1
        } else {
            j = next[j];  // 失配时用 next 回溯 j
        }
    }
}
```

**📌 KMP 改进（nextval）：**
- 当 `T[i] == T[next[i]]` 时，`next[i]` 的跳转是无效的 → 递归修正
- `nextval` 进一步减少了无意义的比较次数

**项目中的 KMP 代码包含了详细的**：
- 手动推导示例（`T = "abcac"` 的 next 数组演变）
- `KMP_Stats` 统计结构体（比较次数、回溯次数、总步数）
- `nextval` 数组改进前后的效率对比

### 第6章：树与二叉树 ★★★★★

这是全书最重要的章节，也是 408 分值最高的章节。

#### 二叉树的存储

| 存储方式 | 适用场景 | 优缺点 |
|----------|----------|--------|
| **顺序存储** | 完全二叉树 | 空间浪费（一般二叉树） |
| **二叉链表** | 通用 | lchild + rchild 双指针 |
| **三叉链表** | 需访问双亲 | 增加 parent 指针 |
| **线索链表** | 快速找前驱/后继 | 利用空指针域 |

#### 二叉树的遍历

**📌 核心思想：**
- **先序**：根 → 左 → 右
- **中序**：左 → 根 → 右
- **后序**：左 → 右 → 根
- **层序**：从上到下、从左到右（借助队列）

**📌 巧妙实现 — 非递归中序遍历：**
```c
// 借助栈：先一路向左入栈，出栈访问，再转向右子树
while (p || !StackEmpty(S)) {
    if (p) {                // 一路向左
        Push(&S, p);
        p = p->lchild;
    } else {                // 出栈访问，转向右
        Pop(&S, &p);
        Visit(p->data);
        p = p->rchild;
    }
}
```

#### 线索二叉树

**📌 核心思想：**
- 利用 n+1 个空链域存放前驱/后继指针
- `LTag=0` → lchild 指向左孩子；`LTag=1` → lchild 指向前驱
- `RTag=0` → rchild 指向右孩子；`RTag=1` → rchild 指向后继

**📌 线索化的关键：**
- 中序遍历过程中修改空指针
- **"当前结点 p 填补前驱 pre 的后继"**—— 这是线索化的核心法则

```c
void InThreading(BiThrTree p) {
    if (p) {
        InThreading(p->lchild);       // 左子树线索化
        if (!p->lchild) {             // p 无左孩子
            p->LTag = Thread;         // 建立前驱线索
            p->lchild = pre;
        }
        if (!pre->rchild) {           // pre 无右孩子
            pre->RTag = Thread;       // 建立后继线索
            pre->rchild = p;
        }
        pre = p;
        InThreading(p->rchild);       // 右子树线索化
    }
}
```

#### 哈夫曼树与哈夫曼编码

**📌 核心算法（算法 6.12）：**
1. 从 n 个叶子结点（权值已知）开始
2. 每次选择权值最小的两个结点合并为新分支结点
3. 新结点的权值 = 两子结点权值之和
4. 重复 n-1 次，得到完整的哈夫曼树

**📌 巧妙实现：**
- 使用 0 号单元未使用的数组 `HT[1..m]`（m = 2n-1）
- `Select(HT, i-1, &s1, &s2)` 每次在 `parent=0` 的结点中选最小两个
- 编码从叶子到根逆向生成：左 0 右 1
- **无栈非递归遍历**哈夫曼树求编码的优化实现

#### 回溯法与树遍历

**📌 典型问题：**
- **幂集**：每个元素选或不选，形成一棵完全二叉树
- **四皇后**：在 n×n 棋盘上放置 n 个皇后，互不攻击
  - 检查条件：列冲突 `x[k] == j`、对角线冲突 `abs(i-k) == abs(j-x[k])`
  - 回溯：当前行找不到合法位置 → 回溯上一行换列

#### 并查集

**📌 408 考点：**
- 用树的双亲表示法实现
- `Find`：找根结点
- `Union`：合并两棵树（可优化：按秩合并 + 路径压缩）

### 第7章：图 ★★★★★

图的章节是全书**最难**、**最丰富**、也是 408 大题最爱考察的章节。

#### 四种存储结构

| 存储结构 | 适用 | 空间复杂度 | 求度 | 408 频率 |
|----------|------|-----------|------|----------|
| **邻接矩阵** | 稠密图 | O(n²) | O(n) | ⭐⭐⭐⭐⭐ |
| **邻接表** | 稀疏图 | O(n+e) | O(n+e) | ⭐⭐⭐⭐⭐ |
| **十字链表** | 有向图 | O(n+e) | O(1) 求入度/出度 | ⭐⭐⭐ |
| **邻接多重表** | 无向图 | O(n+e) | — | ⭐⭐⭐ |

**📌 408 高频对比：**
- 邻接矩阵：唯一确定、随机访问边、空间 O(n²)
- 邻接表：节省空间（稀疏图）、出度易求、入度需遍历

#### 图的遍历

**DFS（深度优先搜索）：**
- **核心思想**：递归（或借助栈），类似于树的先序遍历
- **时间复杂度**：邻接矩阵 O(n²)、邻接表 O(n+e)
- **应用**：判断连通性、找连通分量、拓扑排序、关键路径

**BFS（广度优先搜索）：**
- **核心思想**：借助队列，类似于树的层序遍历
- **时间复杂度**：同 DFS
- **应用**：求无权图的最短路径

**📌 非连通图的遍历：**
```c
void DFSForest(ALGraph G, CSTree *T) {
    for (int v = 0; v < G.vexnum; ++v)
        if (!visited[v])  // 发现新的连通分量
            DFSTree(G, v, &T);  // 生成一棵生成树
}
```

#### 最小生成树

| 算法 | 核心思想 | 时间复杂度 | 适用 |
|------|----------|-----------|------|
| **Prim** | 从顶点出发，每次选最小边 | O(n²) | 稠密图 |
| **Kruskal** | 从边出发，每次选最小边且不形成环 | O(e log e) | 稀疏图 |

**📌 Prim 的实现巧妙：**
- `closedge[]` 数组：记录从 U 到 V-U 的最小代价边
- `lowcost=0` 表示该顶点已并入 U
- 每次在 `lowcost>0` 中找最小值

#### 拓扑排序

**📌 核心思想：**
1. 从 DAG 图中选择入度为 0 的顶点输出
2. 删除该顶点及其所有出边（邻接点入度减 1）
3. 重复直至所有顶点输出

**📌 巧妙实现：**
- 用**栈**暂存入度为 0 的顶点（也可用队列）
- 用 `indegree` 域记录每个顶点的入度
- 每次删除顶点时，其邻接点入度减 1，减到 0 则入栈

#### 关键路径

**📌 AOE 网：**
- `ve[j]` = 事件 j 的最早发生时间（从源点向前推）
- `vl[j]` = 事件 j 的最迟发生时间（从汇点向后推）
- 关键路径 = `ve == vl` 的活动组成的路径

#### 最短路径

| 算法 | 解决 | 时间复杂度 | 限制 |
|------|------|-----------|------|
| **Dijkstra** | 单源最短路径 | O(n²) | 不能有负权边 |
| **Floyd** | 所有顶点间最短路径 | O(n³) | 可处理负权（不能有负环） |

**📌 Dijkstra 巧妙之处：**
- `D[i]` 数组：当前从源点到 i 的最短路径长度
- `P[i]` 数组：路径上 i 的前驱
- 每次选 `D` 中最小的未处理顶点加入 S
- 更新其邻接点的 `D` 值

**📌 Floyd 的核心思想（动态规划）：**
```
D⁰[i][j] = arcs[i][j]       (直接距离)
Dᵏ[i][j] = min(Dᵏ⁻¹[i][j], Dᵏ⁻¹[i][k] + Dᵏ⁻¹[k][j])  (经过 k 是否更短)
```

#### 顺序查找

**📌 巧妙实现 —— 哨兵：**
```c
ST.elem[0].key = key;  // 哨兵
for (i = ST.length; !(ST.elem[i].key == key); --i);
return i;  // 找不到时 i=0
```
- 哨兵避免了每次循环都判断 `i > 0`（越界检查）
- 减少一次比较 → 数据量大时性能提升显著
- 属于**经典优化技巧**

#### 折半查找

**📌 核心思想：**
- 每次将查找区间对半分割
- `mid = (low + high) / 2`
- 判定树的高度不超过 `⌊log₂n⌋ + 1`
- **ASL ≈ log₂(n+1) - 1**

**📌 408 易错点：**
- 循环条件 `low <= high`（取等号！）
- `high = mid - 1` 和 `low = mid + 1`（跳过 mid）

## 🔧 如何运行

### 环境要求

- **C 编译器**：GCC / MinGW / MSVC
- **C++ 编译器**：G++ / MSVC
- **IDE 推荐**：Dev-C++ / Visual Studio / CLion / VS Code

### 编译运行

```bash
# 以单链表为例（C 版）
gcc Chapter02_LinearList/LinkedList/LinkList.c -o LinkList
./LinkList

# 以 KMP 为例
gcc Chapter04_String/PatternMatching/KMP.c -o KMP
./KMP

# C++ 版本
g++ Chapter02_LinearList/LinkedList/LinkList_CPP.cpp -o LinkList_CPP
./LinkList_CPP
```

### 在 Dev-C++ 中运行

1. 打开 Dev-C++
2. `文件 → 打开` 选择对应的 `.c` 或 `.cpp` 文件
3. `运行 → 编译运行`（或按 F11）

### 在 Visual Studio 中运行

1. 创建新的**控制台应用程序**项目
2. 将源文件添加到项目中
3. 按 Ctrl + F5 运行

> **注意：** 项目已统一注释所有 `system("pause");` 语句，如需保留暂停效果可取消注释。

## 📐 代码风格

### 整体风格

```
✅ 教材风格          → Status 返回值、ElemType 泛型
✅ 传统 C 实现       → malloc/free、指针操作
✅ 可读性优先        → 完整的变量命名、中文注释
✅ 教学优先          → 有多份实现、有对比说明
✅ 严谨性优先        → 每个边界条件都检查
❌ 不追求花哨        → 不用宏技巧、不用奇技淫巧
❌ 不依赖 STL        → 所有数据结构手写
```

### 命名约定

| 元素 | 命名规则 | 示例 |
|------|----------|------|
| 类型名 | PascalCase | `SqList`, `LinkList`, `BiTree` |
| 函数名 | PascalCase | `InitList`, `ListInsert`, `GetElem` |
| 变量名 | 小写 + 下划线 | `p`, `q`, `length`, `list_size` |
| 常量 | 全大写 + 下划线 | `LIST_INIT_SIZE`, `OVERFLOW` |
| 状态码 | 全大写 | `OK`, `ERROR`, `TRUE`, `FALSE` |

### Status 返回值规范

每个基本操作返回 `Status` 类型：
- `OK` (1)：操作成功
- `ERROR` (0)：操作失败（参数非法、表为空等）
- `OVERFLOW` (-2)：内存分配失败
- `INFEASIBLE` (-1)：操作不可行

## 💬 注释规范示例

**函数首部注释：**
```c
// 在带头结点的单链表 L 中，删除第 i 个元素，并用 e 返回其值
// 参数：L - 链表头指针, i - 删除位置(1-based), e - 输出参数
// 返回：OK - 删除成功, ERROR - 位置不合法
```

**关键逻辑注释：**
```c
// 下面两步不能交换位置
s->next = p->next;  // 先让新结点指向第 i 个结点
p->next = s;        // 再让第 i-1 个结点指向新结点
```

**边界条件注释：**
```c
// 上述循环结束有两种情况：
// 1. p 到了最后一个节点的指针域(NULL) → i 偏大
// 2. 输入 i 为 0 或负数 → i 偏小
if (!p || j > i) return ERROR;
```
